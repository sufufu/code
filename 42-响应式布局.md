### 响应式布局

根据不同设备的屏幕宽度，开发一套可以适配所有尺寸的页面，并且保证在不同设备上呈现的效果是一样的（响应式布局是一种解决方案）

响应式布局的核心是保证元素尺寸占据设备屏幕的比例相同

+ 常见的响应式布局解决方案
- 媒体查询
- rem 布局
- flex 布局
- vw vh 结合 rem 
- 流式布局

### vh 和 vw

```html
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Title</title>
  <style>
    * {
      margin: 0;
      padding: 0;
    }

    html {
       font-size: 13.3333333vw; !*100px*!
     }
     .box {
       margin: .3rem auto;
       width: 3rem;
       height: 3rem;
       background: lightgreen;
     }

    html, body {
      width: 100%;
      height: 100%;
    }
    .mask {
      position: fixed;
      left: 0;
      top: 0;
      width: 100vw;
      height: 100vh;
      background: rgba(0, 0, 0, .3);
      display: flex;
      justify-content: center;
      align-items: center;
    }
    .mask .content {
      width: 600px;
      height: 400px;
      background: #00b38a;
    }
  </style>
</head>
<body>
<div class="box"></div>
<div class="mask">
  <div class="content"></div>
</div>

<script>
  // vw 和 vh：把视口的宽高各分为100份；每一份宽就是 1vw，每一份高就是 1vh；vw 和 vh 是两个单位；
  // 所以在不同宽度的设备上 1vw 代表的 px 数是不同的；
  // 在 750px 的设备上 1px 是多少 vw？ 750px = 100vw，用 100vw / 750px = 0.133333... vw / px

  // 375px 1px 是多少vw ？ 100vw = 375px；1px 是多少 0.26666 vw；

  // vw 有什么用？
  // 1. 所有样式的尺寸都可以用 vw 来写，但是因为算出来的值大概率不是整数，存在四舍五入的情况，最后容易导致样式差一点；所以一般情况下还是用vw 结合 rem 来用；

  // 2. 让一个盒子充满屏幕: 把这个盒子的宽设置 100vw，高设置100vh；

</script>
</body>
</html>
```
